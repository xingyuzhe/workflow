# SSOT — artifact ownership (Workflow v2)

| Content | Authority |
|---------|-----------|
| Why / scope / capabilities | `proposal.md` |
| Cross-cutting decisions | change-level `design.md` |
| Module structure / types | `specs/<capability>/design.md` |
| Verifiable behavior | `specs/<capability>/spec.md` |

Rules also live in `openspec/config.yaml` (overwritten by init) and `workflow-spec` schema instructions.

Never duplicate the same multi-line structural content in two artifacts — link to the SSOT instead.
